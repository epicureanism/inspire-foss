Technical Features

Deegree version 3 supports complex ISO GML Application Schemas as used within INSPIRE data themes. Listed below are deegree 3 features relevant for INSPIRE data support.

 

deegree 3 WFS Features

  • GML 3.2.1 output (GML 2, 3.0, 3.1 is also available)
  • WFS 1.1.0 protocol (1.0.0 also available, 2.0.0 support planned)
  • Fully transactional (inserting of unmodified complex INSPIRE data sets supported)
  • High performance and excellent scalability due to streaming architecture
  • Queries may use complex filter expressions that take advantage of the full power of XPath 1.0
  • Simple configuration based on unmodified INSPIRE data schemas
  • Query XLink support: output GML features can be references or expanded
  • Backend storage support: PostGIS (work in progress), Oracle Spatial (work in progress) or GML instance documents.
  • The pluggable and modular data access layer allows to add support for new backends with ease.

deegree 3 WMS Features

  • Styling and rendering of unmodified (complex) INSPIRE features
  • Supports SLD and SE (Symbology Encoding) 1.0 standards for styling
  • GetFeatureInfo returns full INSPIRE GML objects
